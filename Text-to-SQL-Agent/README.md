# AI Credit Risk Data Assistant 🤖

## Overview
This project is an AI-powered Text-to-SQL agent built to democratize data access within a credit risk department. It allows risk managers, underwriters, and product teams to query a credit risk database using natural language, significantly reducing the bottleneck on the data analytics team.

Unlike standard conversational AI, this agent does not rely on its internal memory (preventing hallucinations). Instead, it translates natural language questions into precise SQL queries, executes them against a local SQLite database, and returns accurate insights based on the actual data.

## Business Value & ROI
* **Efficiency:** Automates routine data extractions and basic aggregate reporting.
* **Accuracy:** Executes deterministic SQL queries ensuring 100% data integrity.
* **Transparency:** The UI explicitly reveals the underlying SQL query generated by the LLM, allowing data professionals to audit the logic and ensure accurate calculations (e.g., Default Rates, Total Exposure).

## Tech Stack
* **UI & Frontend:** Streamlit
* **Orchestration:** LangChain (`create_sql_agent`)
* **LLM:** OpenAI API (GPT-4o-mini)
* **Database:** SQLite & SQLAlchemy
* **Data Manipulation:** Python & Pandas
  
## How to Run Locally
1. Clone the repository: `git clone https://github.com/SimonaLevitsky/Data-Projects.git`
2. Navigate to the specific project folder: `cd Data-Projects/Text-to-SQL-Agent`
3. Install the required dependencies: `pip install -r requirements.txt`
4. Add your OpenAI API key securely to your environment variables.
5. Run the Streamlit application: `python -m streamlit run app.py`

*Note: The included `credit_risk.db` is a mock database generated strictly for demonstration purposes.*
